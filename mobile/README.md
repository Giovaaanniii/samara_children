# Самара Детям — мобильное приложение (Expo + React Native)

Клиент к тому же **FastAPI** бэкенду, что и веб-сайт (`/api/v1`). Реализованы: главная с «Популярными», каталог с фильтрами и пагинацией, карточка мероприятия и сеансы, бронирование с оплатой (ссылка ЮKassa открывается в браузере), вход и регистрация, профиль со списком бронирований, блок «О нас».

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Укажите `EXPO_PUBLIC_API_BASE_URL` (как `VITE_API_BASE_URL` у фронта в корне `samara/.env`):
   - **Android-эмулятор** → хост-ПК: `http://10.0.2.2:8000/api/v1`
   - **Тот же ПК / iOS-симулятор**: `http://127.0.0.1:8000/api/v1`
   - **Физическое устройство в Wi‑Fi**: `http://<IP_ПК>:8000/api/v1` (в бэкенде должен быть разрешён CORS для этого origin нет — в React Native CORS браузера нет; нужен доступ по сети к порту 8000).
3. Опционально: `EXPO_PUBLIC_WEB_BASE_URL=http://localhost:3000` — ссылки «Оферта / Политика» в разделе «О нас» откроются в браузере.

## Запуск

```bash
cd mobile
npm install
npx expo start
```

Дальше отсканируйте QR в **Expo Go** или нажмите `a` / `i` для эмулятора.

## Роль «гид»

Вкладка **Профиль** после входа под учёткой гида показывает тот же набор, что на сайте: редактирование профиля, **рейтинг от клиентов**, **расписание** (фильтры по периоду и статусу сеанса), подтверждение/отказ, завершение сеанса, **подтверждение по номеру брони**, просмотр **состава группы** по брони.

## Отличия от сайта

- Нет админ-панели в приложении (для админа показано сообщение — использовать сайт).
- Нет входа через VK (нужен `WebBrowser` / deep linking и настройка redirect).
- Юридические страницы при наличии `EXPO_PUBLIC_WEB_BASE_URL` ведут на сайт.
